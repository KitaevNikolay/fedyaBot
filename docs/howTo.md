# 📘 Руководство пользователя FedyaBot

FedyaBot — это инструмент для автоматизированной генерации контента на основе исходных файлов `.docx`. Бот помогает создавать статьи, проводить факт-чекинг, выполнять SEO‑оптимизацию и генерировать описания продуктов и рубрик.

## 🚀 Начало работы

### Основные команды
- `/start` — Запустить бота и начать новую сессию.
- `/cancel` — Отменить текущее действие и сбросить состояние (полезно, если что-то пошло не так).

Эти команды доступны в меню ввода сообщений Telegram (наберите `/` или нажмите кнопку "Меню").

---

## 🛠️ Процесс генерации контента

Работа с ботом строится поэтапно. Бот подсказывает следующий шаг с помощью **Roadmap** (дорожной карты) в сообщениях.

### 1. Загрузка исходного файла
1. Введите команду `/start`.
2. Бот попросит отправить файл в формате `.docx`.
3. Загрузите файл. Бот проанализирует его содержимое и выделит вопросы/темы для статьи.

### 2. Подтверждение вопросов
1. После анализа файла бот пришлет список выделенных вопросов.
2. **Действие**: Нажмите кнопку **"Все верно, продолжить"**.
   - *Примечание*: Эти вопросы будут использованы как контекст для генерации статьи.
   - Если вопросы выделены некорректно, нажмите `/cancel` и попробуйте отредактировать исходный файл.

### 3. Генерация статьи
*Статус Roadmap: `Следующий шаг: Генерация статьи`*

1. После подтверждения вопросов появится меню работы со статьей.
2. Нажмите кнопку **"Сгенерировать статью"**.
3. Бот начнет процесс генерации (это может занять некоторое время).
4. **Результат**: Бот пришлет файл со сгенерированной статьей (`.docx`).

### 4. Факт-чекинг
*Статус Roadmap: `Следующий шаг: Факт-чек`*

1. После генерации статьи станет доступна кнопка **"Сгенерировать факт-чек"**.
2. Нажмите её, чтобы проверить факты в статье.
3. Бот выполнит поиск в интернете и проверит утверждения.
4. **Результат**: Бот пришлет файл с отчетом о проверке фактов.
   - *Важно*: Факт-чек сохраняется автоматически, подтверждение не требуется.

### 5. SEO-оптимизация
*Статус Roadmap: `Следующий шаг: Генерация продуктов\подрубрик`*

Этот этап доступен **только после** генерации факт-чека.

1. Нажмите кнопку **"SEO оптимизация"**.
2. Бот попросит прислать ТЗ в формате `.docx`.
3. Отправьте файл с ТЗ. Внутри должны быть:
   - Мета-теги
   - Структура заголовков
   - Список ключевых фраз
   - Ограничения по плотности и дистанциям
4. **Результат**: Бот пришлет файл с SEO‑оптимизированной статьей и покажет расход токенов.

### 6. Генерация продуктов и рубрик
*Статус Roadmap: `Следующий шаг: Генерация продуктов\подрубрик`*

Этот этап доступен **только после** генерации факт-чека.

1. В меню появятся новые кнопки:
   - **"Сгенерировать продукты"**
   - **"Сгенерировать рубрики"**
2. Выберите нужный вариант.
3. **Результат**: Бот пришлет соответствующие файлы с описаниями.

## 💡 Полезные советы

- **Время сессии**: Сессия активна в течение **3 часов**. Если вы не совершали действий дольше, бот может "забыть" контекст. В таком случае начните заново с `/start`.
- **Ошибки**: Если бот не реагирует, попробуйте команду `/cancel` и повторите действие.
- **Токены**: После каждой генерации бот сообщает количество потраченных токенов (стоимость операции).

## ❓ Часто задаваемые вопросы

**В: Я не вижу кнопки "Сгенерировать продукты".**
О: Эта кнопка появляется только после успешной генерации факт-чека. Следуйте по шагам Roadmap.

**В: Как начать работу с новым файлом?**
О: Просто введите команду `/start` в любой момент.
