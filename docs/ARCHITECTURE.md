### Модуль Bot

Содержит логику обработки команд и callback-ов Telegram через **grammY Webhooks**.
Включает в себя:
- **Middleware дедупликации**: Проверка `update_id` через Redis (`SET NX`) для предотвращения повторной обработки.
- Обработку сценариев
- Меню работы со статьями
- Загрузку вопросов, факт-чека и SEO‑ТЗ из .docx файлов
- Отмену ввода через /cancel с возвратом в меню статьи
- Меню администратора (для роли admin)
- Логирование действий через `AppLoggerService` в файлы на хосте.

### Безопасность

- **Webhooks Security**: Проверка входящих запросов по секретному токену (рекомендуется в будущем).
- **AdminGuard**: Guard для проверки роли администратора (в common/guards).
- Проверка админ-доступа основана на поле role у пользователя.
- Валидация входных данных через DTO и Pipes.

## Хранение данных

- **MySQL**: Основная БД (пользователи с профилем Telegram, сессии, статьи, дополнения, настройки генерации).
- **Redis**: Хранение временных состояний бота, контекста генерации и **замков дедупликации** (TTL 1 минута для замков, 3 часа для сессий).
- **Outline**: Хранилище промптов (system/user) с доступом по ID.
- **Логи**: Файловое логирование пользовательских действий, внешних запросов и ошибок генерации. Логи монтируются из контейнера в папку `./logs` на хосте.
- **Bothub логирование**: Для внешних запросов логируются используемые плейсхолдеры system/user промптов.
