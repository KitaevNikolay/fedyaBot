# Changelog

## 0.5.1 - 2026-03-01

### Fixed
- Исправлена ошибка `PrismaClientKnownRequestError` (P2000) при создании статьи с длинным заголовком (более 191 символа). Тип поля `title` в модели `Article` изменен на `Text`.
- Улучшена стабильность запуска приложения: при отсутствии `TEXT_RU_API_KEY` выводится предупреждение в логах вместо аварийной остановки. Ошибка отображается пользователю только при попытке запустить проверку.
- Исправлена ошибка в API text.ru: параметр `text_uid` заменен на `uid` для корректного получения результатов проверки.
- Исправлена ошибка компиляции в `AppModule` из-за некорректного импорта `GenerationSettingsService`.
- Исправлено сохранение ID задачи Bitrix24 в `ArticleAddition` (ранее ID не записывался в БД).

### Added
- Новая команда `/menu` для быстрого вызова меню текущей статьи.
- Интервал опроса API text.ru увеличен с 30 до 45 секунд для снижения нагрузки.

## 0.5.0 - 2026-02-27

### Added
- Интеграция с Bitrix24 для создания задач на размещение статей.
- Поле `bitrixId` в модели пользователя для привязки аккаунта Bitrix.
- Функционал привязки `bitrix_id` через главное меню бота.
- Кнопка "Создать задачу" в меню статьи (доступна после факт-чека).
- Автоматическая загрузка сгенерированных файлов (статья, рубрики, продукты) в задачу Bitrix24.
- Сохранение ID созданной задачи в базе данных (`ArticleAddition` тип `BITRIX_TASK`).
- Очистка сессии и возврат в начало после успешного создания задачи.

## 0.4.2 - 2026-02-27

### Added
- Интеграция с сервисом text.ru для проверки уникальности текста.
- Таблица `TechnicalArticleAdditions` для отслеживания асинхронных технических задач.
- Кнопка "Проверить текст на уникальность" в меню статьи.
- Автоматический опрос (polling) API text.ru для получения результатов проверки.
- Отображение статуса и процента уникальности в интерфейсе бота.
- Механизм обнаружения устаревших проверок (если статья была изменена после проверки).

### Fixed
- Исправлены ошибки типизации при работе с новыми типами дополнений (`SEO_TZ`, `ARTICLE_UNIQ_CHECK`).
