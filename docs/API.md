# API

## Telegram Bot

- Команда `/start` (доступна в меню) создает сессию и запускает сценарий.
- Команда `/cancel` (доступна в меню) сбрасывает текущее состояние.
- **Roadmap**: В сообщениях отображается следующий ожидаемый шаг (Генерация статьи -> Факт-чек -> Продукты).
- Генерация факт-чека и переписывание статьи происходят автоматически с сохранением результата.
- SEO‑оптимизация запускается после факт-чека через загрузку ТЗ в формате .docx.
- Кнопки генерации продуктов и рубрик появляются только после завершения факт-чека.
- Поддерживается загрузка `.docx` файлов для контекста.
- Админ-меню доступно только пользователям с role=admin.
- Сообщения о рубриках и продуктах краткие, без дублирования контента.
- Ошибки генерации логируются в файл с типом generation_error.
- В Users сохраняются telegramId, username, firstName, lastName.
- В логах Bothub фиксируются плейсхолдеры, используемые в system/user промптах.

## Внутренние настройки генерации

- Настройки генерации и промпты управляются через БД и Outline
- Отдельных HTTP эндпоинтов пока нет
